Kafka 基本介紹

Overview

整個 Kafka Cluster 的基本觀念大概可以用下面兩張圖表示:

  1. 一張詳細畫出 Kafka Cluster 內部各個重要元件的關係，以及整個資料的流向。

  2. 另一張我們以媒體生態關係來生動的描述各個元件的互動以及關聯，以便讓看的人很快就可以透過對照的方式知道 Kafka Cluster 內外部的運作。

Broker

解說

Broker 在Kafka Cluster 中的角色就好比整個媒體業的各個新聞台，主要負責所有訊息的傳遞，並且Broker具備的特性還跟台灣媒體十分相似的。

  1. 在台灣，各家媒體沒有人能夠獨霸一方並且在不同報導說法上都有其領導地位（比如水果新聞的辛辣性），Brokers 也是一樣並沒有所謂的 master的概念，每個 broker 依照不同topic 的不同partition擔任leader角色。
  2. 如同新聞台一樣，報導跟說法是由記者提供給新聞台並且報導跟說法的閱讀是由鄉民主動選擇觀看哪一家新聞台的，同理 Broker 對於 Producer 或 Consumer 都是採取被動的方式，等待 Client 端生產或消費訊息而非主動推送或要求訊息，這個設計也再一定程度上保護 Client 端的服務，避免瞬間過大的請求讓Client端服務不堪負荷。
  3. 整個媒體業不會因為一家新聞台倒閉而沒有新聞和報導可以看，相反的剩餘的新聞台反而還會搶分倒閉新聞台的市場，同理 Kafka Cluster 透過設定和配置可以達到服務高可用，不會因為一台 Broker 停止服務導致整個集群不可用，並且剩餘的工作也會由剩下的Broker處理。

摘要

  1. No master cluster
  2. 負責處理Client端的請求
  3. 可以保存訊息直至過期

Producer

解說

Producer 在Kafka Cluster 中的角色就好比整個媒體業的記者，主要負責所有訊息的生產。

  1. 如同記者提供新聞報導和說法一樣，Producer 主要負責提供訊息給 Kafka Cluster。
  2. 如同現在部分新聞報導會有懶人包一樣，Producer 也提供訊息壓縮的機制。
  3. 如同記者提交新聞報導會跟新聞台確認已經收到這篇報導，Kafka 也有 ACK 機制確保Producer傳送的訊息有被Kafka所接收。

摘要

  1. 負責推送訊息至Kafka
  2. 擁有以下幾種推送訊息的策略，使得訊息被分配到對應的Broker上
    • Round-robin
    • Randomness
    • Key-ordering
  3. 擁有以下幾種壓縮訊息的機制
    • GZIP
    • Snappy
    • LZ4
    • Zstandard
  4. ACK

Consumer

解說

Consumer 在Kafka Cluster 中的角色就好比整個媒體業的鄉民，主要負責所有訊息的消費。

  1. 如同我們主動選擇收看哪一家新聞台一樣，Consumer 也是主動向 Broker 拉取訊息。
  2. 如同我們有時候新聞的來源是親朋好友一樣，有時我們不需要實際收看該報導便能知道相關訊息，Consumer 也有 Group 的概念，同一個Group對同筆訊息只會消費一次。
  3. 如同我們自己會記得看過哪一篇新聞報導一樣，Consumer 也有 Offset 的機制會在 Kafka Cluster 中記錄每組 Group 的消費記錄，同時也像我們可以選擇消費的起始點。

摘要

  1. 從Kafka拉取訊息
  2. Consumer group 的優勢
    • 加速消費
    • 高可用
  3. Consumer offsets 的好處
    • 可以指定消費位置
    • 單筆訊息可以多次消費，不會被消費一次後就消失

Topic

解說

Topic 在Kafka Cluster 中的角色就好比整個媒體業的新聞報導，主要是儲存跟報導相關的訊息。

  1. 如同我們有時候會在兩家不同新聞台看到類似的報導和說法一樣，Topic 具有副本的機制，可以幫助我們確保消息的可靠性。
  2. 如同每家新聞台對每種報導和說法各有所長，因此新聞台對於自己熟悉的報導說法一定會在第一時間點取得資訊，其他新聞台在透過各種方式從該新聞台取得該資訊。同理，Topic 有 Partition 的機制會使得訊息首先進入到某台Broker 的某個 Partition 上，若有副本則會從這台Broker上複製到另一台上。

摘要

  1. Partition 有什麼好處:
    • 分散流量
    • 加速消費
  2. Replica 作用:
    • High Availability
  3. 訊息是如何被保存的？
    • 以文件的刑事記錄在最後
    • Producer和不同Consumer間的操作不會互相影響，如下圖所示

Zookeeper

解說

Zookeeper 在Kafka Cluster 中的角色就好比整個媒體業的NCC(管理單位)，主要負責協助 Kafka 管理集群。
如同NCC會提供媒體業一套產業準則，Zookeeper 也會負責儲存Kafka Cluster 相關資訊，其中包含Broker、Topic、Consumer Group 等資訊，具體可看下圖。

摘要

  1. 負責管理 Kafka Cluster 的 metadata
  2. 協助 Controller 管理 Kafka 集群

How to install

Install Zookeeper Cluster

  • Install JAVA
  • Install Zookeeper
  • Set myid for each node
  • Start Zookeeper Cluster

Install Kafka Cluster

  • Install JAVA
  • Install Kafka
  • Set server.properties
  • Start Kafka Cluster
